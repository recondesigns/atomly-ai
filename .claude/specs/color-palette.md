# Color Palette — Foundation Spec

> Foundation spec (not a component). Defines the target primitive color ramps and semantic
> re-aliasing for the "AI-native" palette overhaul. Consult before editing the Design tokens
> Figma file, `scripts/sync-figma-tokens.mjs`, or any theme color values.

## Overview

Replace the stock-Tailwind palette (blue primary, amber brand, green/red functional) with a
violet-tech direction consistent with modern AI products (Linear, Notion AI, Copilot, Gemini):

- **Indigo** becomes the `primary` ramp (replaces blue — blue ramp is removed)
- **Violet** is a net-new `ai` accent ramp, reserved for AI moments (`thinking`, assistant
  accents, "generated by AI" affordances, gradient endpoints) — never used as a second primary
- **Coral** is a net-new `brand` ramp (replaces amber-as-brand)
- **Amber** is demoted to functional `warning` only (a new intent — no warning tokens exist today)
- **Emerald** replaces the neon green as `success`
- **Red** keeps its hues; ramp gaps are filled
- **Neutral (slate 0–1200), white/black, and alpha ramps are unchanged**

**Figma frame:** [Design tokens → Color](https://www.figma.com/design/GFCI1ypTw7ZXlKJzrfGKdz/Design-tokens?node-id=20-2)

## Ramp structure rules

- Every chromatic ramp has the standardized steps **50–950**
  (50, 100, 200, 300, 400, 500, 600, 700, 800, 900, 950). Today's ramps are inconsistent
  (blue 100–900; amber/green/red have 6 gappy steps each).
- The **950 step exists for dark mode**: `*-subtle` backgrounds in dark mode alias 950
  (e.g. `background/primary-subtle` → `indigo.950`). Ramps that stop at 800/900 force dark
  mode to reuse steps that are too light.
- The **600 step is the anchor** for filled backgrounds in light mode (matches current
  convention: `background/primary` → `{blue.600}` becomes `{indigo.600}`).
- White text on any 600 step must pass WCAG AA (4.5:1) — verify when adjusting hexes.

## Impact on code (do not change until Figma + sync are done)

- `scripts/sync-figma-tokens.mjs` reads primitive names (`blue/600`, `amber/500`, `green/600`,
  …) — every `get('blue/...')`-style key must be updated to the new ramp names, or values
  silently fall back to hardcoded (stale) hexes.
- `theme.types.ts` / `defaultTheme.ts` gain `warning*` color slots; component intent unions
  (`ButtonIntent`, `BadgeIntent`, chip intents in `@atomly-ai/types`) may gain `'warning'`.
- Hardcoded fallback hexes in `*.styles.ts` (Button, Badge, Chip) must be updated to the new
  values — Badge's `intentColors` map is entirely hardcoded today (see audit, 2026-07-05).
- Storybook a11y checks run as `test: 'error'` — contrast regressions will fail CI.

## Migration order

1. Update ramps + semantic aliases in the Design tokens Figma file (Token Studio collections)
2. Export via Token Studio → `tokens/figma-export.json`, run `/sync-tokens`
3. Update `sync-figma-tokens.mjs` mapping keys + theme types (warning intent)
4. Update component hardcoded fallbacks; visual pass in Storybook light + dark
